# Кэш дашборда при переключении коллекций

Дата: 2026-07-18

## Цель

Повторный заход на уже загруженную коллекцию (фильтр/поиск) — мгновенно, без мигания всей сетки. Crossfade/затемнение всей сетки не использовать для навигации. Точечно обновлять только то, что реально изменилось при фоновой проверке или правке карточки.

## Почему ломалось раньше

`#176` держал предыдущую сетку и делал dim + commit + fade на **каждый** клик по сайдбару. Снимок на диск (`#78`) — один слот: A→B затирает A. Повторный заход всегда «с нуля» в UI.

## Решение

Session LRU-кэш выборок дашборда + мгновенный показ при попадании + тихая перепроверка с точечным диффом. Скелетон + алерт только когда этой выборки в памяти ещё нет.

## Решения

| Тема | Выбор |
|------|--------|
| Кэш | Память сессии, LRU (~12–20 ключей) |
| Ключ | `filterKey` + `search` (тот же пайплайн для папок/тегов/all/поиска) |
| Повторный заход | Сразу из кэша; фон — перезапрос; без изменений — noop |
| Первый заход | Скелетон в сетке/таблице + «Загрузка…» в стеке алертов |
| Дифф | Стабильный `key={item.id}`; не remount неизменённых; обложку не сбрасывать при том же path/src |
| Таблица | Те же правила; обложек в UI нет — на decode не ждать |
| Навигация | Без dim/спиннера поверх контента и без fade всей сетки |
| Правка карточки | `refreshVault` крутится уже на деталке (хук в AppLayout); тихий refetch + точечный патч, не скелетон |
| Снимок на диск `#78` | Только cold start при запуске приложения, не навигация A↔B |

## Поведение

### Данных этой выборки нет

1. Не показывать карточки другой коллекции.
2. Скелетон сетки или строк таблицы (не пустой экран, не спиннер по центру контента).
3. Статус загрузки — в стеке алертов (как индексация).
4. После готовности окна — сразу контент, без fade «из нуля».
5. Записать в LRU.

### Данные есть

1. Сразу показать кэш.
2. В фоне перезапросить ids/тела/пути.
3. Совпало — ничего не трогать.
4. Нет — убрать/вставить/переставить только дельту; пересечение id остаётся смонтированным.

### Правка / удаление / создание единицы

- Пока пользователь на карточке деталки, хук дашборда уже живёт в `AppLayout` — перезагрузка списка идёт там, не «когда вернёшься на сетку».
- Текущий экран/кэш: точечно (тело карточки, исчезновение из фильтра, появление новой).
- Удаление: выкинуть id из всех записей LRU.
- Прочие ключи LRU: при следующем заходе сразу кэш + тихая проверка (не принудительный скелетон).

## Куда лезть в коде

1. Новый модуль LRU (например `src/services/dashboard-query-cache.ts`).
2. `useDashboardItems`: hit/miss до paint; убрать навигационный `isRefreshing` dim и bump epoch на смену фильтра.
3. `DashboardItemsTransition` / сетка/таблица: скелетон вместо центр-спиннера; загрузка — `AlertStack`.
4. `ItemGridCard`: не сбрасывать ready/pulse, если cover src не изменился.
5. Disk snapshot — seed LRU при старте, если ключ совпал.

## Вне скоупа

- Новые зависимости (React Query и т.п.).
- Несколько смонтированных деревьев дашборда.
- Переделка выбора хранилища в настройках (не часть этой задачи).

## Критерий готовности

- A→B→A: второй заход на A без мигания всей сетки и без скелетона, если A уже грузили.
- Первый заход на неизвестную выборку: скелетон + алерт, не чужие карточки.
- Фоновая проверка без изменений — сетка не дёргается.
- После правки на деталке возврат на сетку без ритуала «как будто коллекцию открыли впервые».
- Таблица: те же hit/miss/дифф, без ожидания обложек.
